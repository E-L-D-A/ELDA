## Adoption on-ramp

The grades ladder assumes a classified tree, so a consumer's payoff today starts only after the rename that gives every file its layer name: no rule bites before it, and a freshly surfaced domain stays unprotected while its consumers are still unsorted. The entries here move payoff earlier while keeping every claim honest per META.6 - each step is a declared narrow scope at full severity, and severity softening inside a rule stays the illegal form the Soundness pass removes. The **Decision** marker carries the Soundness pass convention.

- **Target-side surface integrity.** **Decision.** The plugin README holds "an unsorted file is judged by nothing - every rule declines it". The invariant is right for layer-dependent judgments, where a failed layer claim leaves nothing to judge the file's own conduct against, and over-broad for boundary ones: a reach past a declared surface needs only the target's boundary and the consumer's graph position, and the ownership walk assigns chains to unsorted files too. (a) Re-scope the invariant to the layer-dependent rules and judge the reach-past finding for any consumer the graph places outside the boundary, so protection arrives with each renamed domain and a rename pays at the domain that made it - priced as a two-clause invariant and findings appearing on files that carry no layer claim. (b) Keep the blanket invariant - the simplest classification story, priced as the whole-tree rename staying the precondition for any surface protection.

- **A pre-classification preset.** The rules that judge without layer names - vocab-gate and ambient-ownership, both driven by declared options rather than file classification - ship as their own preset at full severity, with the cycle audit riding beside them as the graph half (reference cycles compute on any resolvable tree; boundary grading just reads narrower while surfaces are sparse). A consumer gets a biting, honestly scoped gate before a single rename: "aligned over the vocabulary and ambient seams" is the coherent narrow claim the grades section blesses. One interaction to accept: the `no-deep-side-effects` unit fix (Per-file rules) moves that rule from directory reading to name reading, which takes it out of this set. Trigger to build: the first consumer tree that wants the gate before its rename.

- **A baseline ratchet for adopting.** The adopting posture's gate is "no new findings in touched files against the accepted baseline", and the preset alone cannot hold it: warn tier exits zero and the comparison stays manual. Build the mechanism - a committed baseline file plus a check that exits non-zero on findings the baseline does not carry - so adopting bites on new code from day one. Same artifact shape as the accepted-cycles file (Graph pass); build them on one mechanism.

Two loose ends ride with the build: the pass stamps a per-cycle gate marker nothing consumes, which the check command wires or the scan drops; and the spec's Three-tiers list files this audit under scheduled maintenance, a residence the check's decidability outgrew the day the pass landed - the filing re-words now to declare the gap and re-files under the machine tier when the gate ships (see Soundness pass).

Trigger to build: the first boundary-drawing session that has to reconstruct the emergent clusters by hand.

The shape half has no home, and the binding walk already resolves which names each consumer takes through each surface, so the graph can report what a reachability tool cannot express: a barrel that publishes a dozen names whose consumers collectively take two, or a named surface with one consumer taking one name, where the boundary is ceremony rather than a contract. Trigger to build: the first surface whose breadth is argued in review with no evidence to hand.

## Visualizer

The diagram reads the current tree: per-edge legality, the cycles the flows close, the closure a pinned file reaches in either direction, and the whole board at whatever scale a domain is folded to. What it does not read is the change.

- **Structural diff against a baseline ref.** Scan a git ref in a detached worktree through the same code path (the walk is path-based, so the scan itself needs no change), then diff files, edges and verdicts: new edges drawn hot, removed edges ghosted, newly classified files outlined. The gate stops the illegal delta at authoring time and stays silent on the legal one, and structural erosion is a sequence of individually legal moves - a fresh cross-domain edge that every rule permits and nobody would have approved on purpose. The unit a person reviews is the change, and the unit the diagram draws is the state; this closes that gap. Trigger to build: the first structural regression found after it landed, or the first review that reconstructs a change's structural delta by hand.
